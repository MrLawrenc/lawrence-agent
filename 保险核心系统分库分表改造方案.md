## 1. 项目背景与目标

### 1.1 项目背景
- 当前系统为单体架构，采用MySQL数据库
- 需要支持千万级别保单，相关数据可能达到上亿甚至10亿级别
- 现有架构无法满足高并发、大数据量的业务需求

### 1.2 改造目标
- 支持千万级保单数据存储
- 提升系统并发处理能力
- 保证数据一致性和系统可用性
- 实现平滑迁移，最小化业务影响

### 1.3 性能指标
- 保单数量：支持1000万+
- 并发用户：支持1000+并发
- 响应时间：核心接口<100ms
- 数据一致性：强一致性保证

## 4. 分库分表设计方案

### 4.1 总体设计原则
- **数据均匀分布**: 确保各分库分表数据量相对均衡
- **最小化跨库操作**: 减少跨库查询和事务
- **可扩展性**: 支持后续水平扩展
- **业务连续性**: 保证改造过程中业务不中断

### 4.2 分库策略
#### 4.2.1 租户分库方案
```
分库规则：按租户ID进行分库
- 租户ID范围：1-1000
- 分库数量：[待确定，建议10-20个]
- 分库算法：hash(租户ID) % 分库数量
```

#### 4.2.2 数据库分布设计
| 数据库类型 | 数量 | 用途 | 存储内容 |
|-----------|------|------|---------|
| 核心业务库 | [待定] | 保单、批改等核心数据 | 保单相关表 |
| 财务库 | [待定] | 财务相关数据 | 财务相关表 |
| 理赔库 | [待定] | 理赔相关数据 | 理赔相关表 |
| 再保库 | [待定] | 再保险相关数据 | 再保相关表 |

### 4.3 分表策略
#### 4.3.1 业务分表方案
```
分表规则：按业务特征分表
- 时间维度：按月/季度/年分表
- 数据特征：按保单号范围分表
- 访问模式：按查询频率分表
```

#### 4.3.2 具体分表设计

| 数据库(租户) | 表类型 | 表名 | 分片键 | 分表算法 | 分表数量 | 备注 |
|-------------|-------|------|-------|---------|---------|------|
| 租户库1-10 | 保单主表 | policy_info | 保单号 | hash(保单号) % N | 100 | 核心保单数据 |
| 租户库1-10 | 保单附件 | policy_attachment | 保单ID | hash(保单ID) % N | 100 | 与保单主表同分片 |
| 租户库1-10 | 保单历史 | policy_history | 保单ID | 按月分表 | 12/年 | 历史数据归档 |
| 租户库1-10 | 批改申请 | endorsement_apply | 保单ID | hash(保单ID) % N | 100 | 与保单同库 |
| 租户库1-10 | 批改历史 | endorsement_history | 保单ID | 按月分表 | 12/年 | 批改历史记录 |
| 租户库1-10 | 批改审核 | endorsement_audit | 状态+时间 | 按状态分表 | 10 | 按审核状态分表 |
| 租户库1-10 | 理赔案件 | claim_case | 理赔号 | hash(理赔号) % N | 200 | 独立分片策略 |
| 租户库1-10 | 理赔明细 | claim_detail | 案件ID | hash(案件ID) % N | 200 | 与案件表同分片 |
| 租户库1-10 | 理赔支付 | claim_payment | 案件ID | 按月分表 | 12/年 | 支付记录按月 |
| 租户库1-10 | 账务主表 | account_main | 账期 | 按账期范围 | 50 | 按财务账期分表 |
| 租户库1-10 | 支付记录 | payment_record | 账期+时间 | 按月分表 | 12/年 | 支付流水按月 |
| 租户库1-10 | 对账记录 | reconciliation | 对账期 | 按对账期 | 12/年 | 对账数据按期 |
| 租户库1-10 | 再保分入 | retrocession_in | 保单ID | hash(保单ID) % N | 100 | 与保单同分片 |
| 租户库1-10 | 再保分出 | retrocession_out | 保单ID | hash(保单ID) % N | 100 | 与保单同分片 |
| 租户库1-10 | 客户信息 | customer_info | 客户ID | hash(客户ID) % N | 50 | 客户基础数据 |
| 租户库1-10 | 产品信息 | product_info | 产品ID | hash(产品ID) % N | 20 | 产品配置数据 |

### 9.1 技术风险
#### 9.1.1 数据一致性风险
- **风险描述**: 分库分表后数据一致性问题
- **影响程度**: 高
- **应对措施**: 分布式事务、补偿机制

#### 9.1.2 性能风险
- **风险描述**: 跨库查询性能下降
- **影响程度**: 中
- **应对措施**: 查询优化、缓存策略

#### 9.1.3 迁移风险
- **风险描述**: 数据迁移过程中的数据丢失
- **影响程度**: 高
- **应对措施**: 多重备份、增量同步

### 9.2 业务风险
#### 9.2.1 业务中断风险
- **风险描述**: 改造过程中业务中断
- **影响程度**: 高
- **应对措施**: 灰度发布、快速回滚

#### 9.2.2 数据准确性风险
- **风险描述**: 分库分表后数据计算错误
- **影响程度**: 高
- **应对措施**: 数据校验、对账机制

### 9.3 运维风险
